# Judicial AI Assistant

A full-stack web application designed to help judges, law clerks, and legal professionals with case management, legal research, document analysis, and courtroom preparation — powered by AI and grounded in real case law data.

## Features

### AI-Powered Legal Tools

- **Case Briefer** — Upload legal documents (including scanned PDFs) and receive structured case summaries generated by AI
- **Legal Research** — Search for real case law via the CourtListener API with jurisdiction, court, and date filters, then get AI-generated analysis grounded in actual precedent
- **Case Law Explorer** — Explore historical cases and legal principles by topic and jurisdiction
- **Evidence Analyzer** — Upload and analyze evidence documents for relevance, admissibility, and patterns
- **Order Drafter** — Generate formal court orders and rulings from case details and ruling parameters
- **Jury Instruction Drafter** — Create jury instructions tailored to specific case facts and legal standards
- **Oral Argument Coach** — Prepare for oral arguments with AI-generated key questions, counterarguments, and coaching tips
- **Daily Docket** — Manage court schedules with Google Calendar integration

### Case Management

- Create and track cases manually or by uploading an initial pleading for automatic AI extraction of case name, number, and court
- Select an active case in the sidebar to provide context across all tools
- Upload and manage case documents with full-text search

### Dashboard

- Overview of active/pending cases and upcoming calendar events
- Quick-access grid to all AI tools
- Activity history tracking across all interactions

## Getting Started

### Prerequisites

- Node.js 18+
- PostgreSQL database

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `OPENAI_API_KEY` | No | OpenAI API key for GPT-4o (recommended for accuracy) |
| `GEMINI_API_KEY` | Yes | Google Gemini API key (used as primary or fallback) |
| `SESSION_SECRET` | Yes | Secret for session encryption |

### Installation

```bash
# Install dependencies
npm install

# Push database schema
npm run db:push

# Start development server
npm run dev
```

The app will be available at `http://localhost:5000`.

### Production Build

```bash
npm run build
npm start
```

## AI Provider Strategy

The application uses a dual-provider approach to maximize accuracy and availability:

1. **OpenAI GPT-4o** (primary when configured) — Lower temperature (0.3) for factual, consistent legal analysis. All prompts are designed to prevent hallucination of case names, citations, or holdings.
2. **Google Gemini** (fallback) — Automatically activated if OpenAI is unavailable or quota is exceeded. Uses the same anti-hallucination prompt engineering.

All AI-generated legal analysis is grounded in actual case law retrieved from CourtListener. The system explicitly instructs the AI to only cite cases that appear in the search results and to clearly distinguish between specific case references and general legal principles.

## Legal Research Pipeline

1. User submits a search query with optional jurisdiction, court, date range, and status filters
2. The CourtListener API returns matching real-world cases with full metadata
3. Case results are passed as context to the AI provider
4. The AI generates a structured research report citing only the cases provided
5. Results include direct links to full case text on CourtListener
